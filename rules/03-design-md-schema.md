# 03 design.md 结构规则

## 目标

规定 `design.md` 的章节结构、信息来源、写作密度和分卷策略。

## 文档定位

`design.md` 是给模型执行的 UI 生成规范，不是完整设计说明书。

它应回答：

- HTML 原型应该使用什么视觉风格；
- 有哪些组件；
- 组件怎么写 class、尺寸、状态和结构；
- 有哪些典型页面模板；
- 不应该生成什么；
- 需要引用哪些截图。

## 推荐章节

```text
1. 规范来源
2. 产品设计气质
3. 原型生成优先级
4. 设计 Token
5. 组件清单
6. 组件契约
7. 页面 Example 样例
8. 补充组件使用规则
9. 布局模式
10. 交互状态
11. HTML 原型生成规则
12. 禁止事项
13. 待补与变更记录
```

## 章节来源标注

每个章节都必须能追溯来源。

| 章节 | 默认来源 | 处理方式 |
| --- | --- | --- |
| 规范来源 | Figma 链接、用户输入 | 自动整理 |
| 产品设计气质 | 用户输入、模型归纳 | 首次必须标记待确认 |
| 原型生成优先级 | 用户输入、默认规则 | 可先写默认值 |
| 设计 Token | Figma 节点 | 自动抽取 |
| 组件清单 | Figma 节点 | 自动抽取 + 分类 |
| 组件契约 | Figma 节点、截图 | 自动生成 + 用户可修订 |
| 页面 Example | Figma 页面节点 | 自动生成 |
| 补充组件使用规则 | Figma 节点、用户说明 | 自动生成 |
| 布局模式 | 页面 example 归纳 | 模型归纳 |
| 交互状态 | 组件契约归纳 | 模型归纳 |
| HTML 原型生成规则 | 默认规则、用户补充 | 默认生成 |
| 禁止事项 | 默认规则、用户补充 | 默认生成 |
| 待补与变更记录 | manifest | 自动同步 |

## 主文档长度控制

`design.md` 应优先保留高频信息。

启动分卷条件：

```text
组件契约总数 > 20
或 design.md 超过 30000 中文字符
或 单个组件契约超过 1200 中文字符
```

分卷后：

```text
design.md 保留：
- 组件清单；
- 高频组件契约；
- 页面 Example；
- 原型生成规则；
- 禁止事项。

references/contracts/ 保留：
- 低频组件长契约；
- 复杂组件变体矩阵；
- 大量截图引用。
```

`design.md` 中用相对路径引用：

```text
Dialog 对话框：详见 references/contracts/Dialog.md
```

## 锚点与节点追踪

写入由 Figma 节点生成的段落时，推荐使用 Markdown 注释作为机器可读锚点。

```markdown
<!-- figma-node: EnNxABhxlnzyZyo9Z7ZsvH#1140:54841 section: 6.17 hash: sha256:{contentHash} -->
正文内容
<!-- /figma-node -->
```

锚点用于增量合并和用户修改保护。若用户不希望主文档出现注释，可只在 manifest 中记录。

## 写作风格

必须：

- 中文；
- 短句；
- 明确 class、尺寸、布局和状态；
- 使用表格承载清单；
- 使用 ASCII 骨架表达页面结构；
- 使用“必须 / 不得 / 优先 / 仅允许”等执行词。

避免：

- “高级感”“丰富层次”“现代化体验”等空泛描述；
- 长篇设计理念；
- 没有 class 或结构约束的组件说明；
- 只有截图没有文字契约；
- 只有文字没有截图依据。
